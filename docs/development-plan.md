# Windows 开发方案

## 目标

在 Windows 平台重新实现 houmao，保持 Mac 版的核心能力，同时遵循 Windows 原生最佳实践，优先做到稳定、轻量、可维护、可发布。

---

## 开发原则

1. 优先原生 Windows 体验，不做跨平台中间层。
2. UI、系统交互、数据存储、网络通信分层清晰。
3. 先 MVP 后增强，避免一开始引入过重的复杂性。
4. 所有系统能力都要有失败退化路径，例如热键、毛玻璃、托盘、UIA 读取。
5. 先把可用性做对，再追求特效和个性化。

---

## 阶段划分

### 阶段 0：仓库与工程骨架

目标是把 Windows 版工程跑起来，建立可扩展结构。

任务：
- 建立 `src/Houmao` WPF 工程。
- 建立 `tests/Houmao.Tests` 测试工程。
- 加入 `App.xaml`、`MainWindow.xaml`、`SettingsWindow.xaml`、基础资源字典。
- 配置依赖注入、日志、配置文件路径、应用生命周期。
- 建立本地开发和发布脚本。

验收标准：
- 项目能编译并启动空窗口。
- 关闭主窗口不会退出进程。
- 托盘图标可显示并可退出。

---

### 阶段 1：主窗口与输入链路

目标是先实现最核心的对话入口。

任务：
- 实现浮动主窗口样式、圆角、阴影、毛玻璃退化背景。
- 实现输入框、回车提交、上下箭头命令历史。
- 实现 `b` / `h` 面板切换命令。
- 实现 `Esc` / `Ctrl+W` 隐藏窗口。
- 实现窗口显示时自动聚焦输入框。
- 实现附件选择与列表展示。

验收标准：
- 用户可输入问题并触发 ViewModel 提交。
- 用户可添加和移除图片/音频附件。
- 用户可通过快捷键切换历史和帮助面板。

---

### 阶段 2：AI 客户端

目标是把请求 LLM 的主链路跑通。

任务：
- 实现 OpenAI 兼容 `chat/completions` 客户端。
- 支持流式输出和取消。
- 支持 `reasoning_content` 的读取与展示。
- 支持 `<think>...</think>` 推理标签过滤（流式状态机 + 非流式正则）。
- 支持多轮对话历史，最多保留 20 条消息。
- 支持 `@provider` 与 `@model` 路由。
- 支持图片与音频附件上传。

验收标准：
- 任意兼容 OpenAI 的 Provider 均可正常对话。
- 取消请求后 UI 状态正确恢复。
- 流式输出逐字显示，不阻塞 UI。
- 含 `<think>` 标签的模型输出能正确隐藏推理过程。

---

### 阶段 3：配置与设置

目标是让用户可自定义 Provider 和行为开关。

任务：
- 实现 Provider 增删改查。
- 实现默认 Provider 置顶逻辑。
- 实现 API Key 安全输入与持久化。
- 实现设置窗口和托盘菜单。
- 实现开机自启、划选即复制开关、主题切换策略。

验收标准：
- 用户可在设置窗口完成完整 Provider 管理。
- 配置修改后可立即生效。

---

### 阶段 4：历史与使用追踪

目标是复刻 Mac 版的使用历史能力。

任务：
- 实现使用记录存储。
- 实现前台应用监控。
- 实现 UI Automation 文本读取。
- 实现键盘缓冲区回退（UIA 失败时降级）。
- 实现应用切换记录（`[Switch] A → B`）。
- 实现历史列表、分页、清空。
- 增加隐私提示与开关。

验收标准：
- 用户在其他应用输入并回车后，能记录应用名与文本片段。
- CJK 输入法文本能正确读取。
- 历史面板可正常翻页和清空。

---

### 阶段 5：全局热键与辅助功能

目标是复刻 Mac 版快捷召唤体验。

任务：
- 实现双击 Alt 唤起窗口。
- 实现防抖和误触保护。
- 实现划选即复制。
- 实现托盘菜单与退出路径。

验收标准：
- 双击 Alt 稳定唤起和隐藏窗口。
- 辅助功能相关能力出现异常时不影响主对话功能。

---

### 阶段 6：打包发布

目标是产出可分发安装包。

任务：
- 配置单文件发布、图标、版本信息。
- 产出安装包或便携版 zip。

验收标准：
- 可生成可直接运行的发布产物。

---

## 建议实施顺序

1. 先做工程骨架和主窗口。
2. 再做 AI 客户端和对话链路。
3. 然后补配置、历史、托盘、热键。
4. 最后做使用追踪和划选即复制。

---

## 关键风险

- 全局输入监听和 UI Automation 属于系统敏感能力，可能受权限和安全软件影响。
- Windows 10 和 Windows 11 的窗口特效 API 不完全一致，需要有明确 fallback。
- 流式输出、取消、窗口切换和输入法之间要特别注意线程切换。
- 使用历史追踪要把隐私提示和关闭开关做清楚。
